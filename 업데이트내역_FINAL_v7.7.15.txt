FINAL v7.7.15 · 발송처리 상품주문 건수 기준

핵심 집계 기준
- 주문 건수는 송장번호 수, 배송묶음 수, 구매자 수로 계산하지 않습니다.
- 실제 상품주문 처리 행을 기준으로 계산합니다.
- 한 구매자가 서로 다른 상품 2개를 합배송하고 송장 1개를 사용해도 상품주문은 2건으로 계산합니다.
- 동일 상품주문 행이 API에서 반복 수집된 경우에만 1건으로 중복 제거합니다.
- 상품 수량이 2개여도 상품주문 행이 1개이면 주문 건수는 1건입니다.

마켓별 상품주문 식별 기준
- 쿠팡: 주문번호 + 쿠팡 상품주문 행(lineKey/vendorItemId)
- 스마트스토어: 상품주문번호(productOrderId)
- 11번가: 주문번호 + 주문상품 순번
- 롯데온: 주문상품 행 식별값
- G마켓·옥션: API 승인 전까지 미연동·집계 제외

화면 반영
- 오늘 주문: 오늘 접수된 상품주문 처리 행 수
- 이번달 총주문: 이번 달 상품주문 처리 행 수
- 쇼핑몰별 오늘 주문: 같은 기준으로 마켓별 합산
- 현재 진행 목록의 신규·발송대기: 같은 송장을 공유해도 서로 다른 상품주문은 각각 표시·집계
- 오늘 매출과 월 매출: 같은 주문 총액이 상품행마다 반복되어도 중복 합산하지 않도록 기존 주문 단위 금액 계산 유지
- 오늘 판매 TOP 20: 상품별 실제 판매수량과 상품금액 기준 유지

스마트스토어 고정 기준
- 선물하기 WAIT_FOR_RECEIVING 미수락 주문은 신규주문·오늘 주문·매출·텔레그램에서 제외
- 수락 후 정상 상품주문으로 반영

마켓 공식 API 직접검증 유지
- 쿠팡·스마트스토어·11번가·롯데온 공식 API 응답을 직접 조회
- 취소·반품·교환·문의 현재 상태 직접 대조
- 쿠팡 교환 RECEIPT/PROGRESS 진행 건 유지
- G마켓·옥션 승인 전 집계 제외

기타 수정
- 마켓직접검증_결과열기.cmd를 한글 인코딩 오류 없이 실행되는 ASCII 방식으로 수정
- 신규주문·취소·반품·교환·문의 텔레그램 상품 썸네일 유지
- TOP 20, 금액 보강, 전체 글자 확대 유지
