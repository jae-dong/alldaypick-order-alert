FINAL v7.4.0 REVIEWED 배포 적용 순서

현재 사용 주소: https://jae-dong.github.io/alldaypick-order-alert/
현재 GitHub 저장소: jae-dong/alldaypick-order-alert

1. 실행 중인 검은 START_AGENT 창을 닫습니다.
2. 이 ZIP을 별도 폴더에 완전히 압축 해제합니다.
3. GitHub Desktop에서 Current repository가 alldaypick-order-alert인지 확인합니다.
4. GitHub Desktop 메뉴 Repository > Show in Explorer를 눌러 실제 연결 폴더를 엽니다.
5. 압축 해제한 폴더 안의 파일과 backend 폴더를 그 연결 폴더에 덮어씁니다.
6. 연결 폴더 안의 .git 폴더는 절대 삭제하거나 덮어쓰지 않습니다.
7. GitHub Desktop으로 돌아가 변경 파일을 확인합니다.
8. Summary에 FINAL v7.4.0 검토배포본 적용 을 입력합니다.
9. Commit to main을 누릅니다.
10. 상단 Push origin을 누릅니다.
11. 2~5분 후 기존 웹주소를 열고 상단 버전이 FINAL v7.4.0인지 확인합니다.
12. 버전이 안 바뀌면 웹앱을 완전히 종료한 뒤 브라우저 사이트 데이터/캐시를 삭제하고 다시 접속합니다.
13. GitHub에 연결된 최종 폴더의 START_AGENT.cmd를 실행합니다.
14. 시작 정밀 동기화가 완료될 때까지 검은 창을 닫지 않습니다.
15. 웹앱에서 지금 수집을 한 번 누르고 3~10분 후 새로고침합니다.

검증 기준
- 신규 -> 발송대기 이동 시 신규 감소, 발송대기 증가
- 발송대기 -> 배송중/배송완료 이동 시 발송대기 감소
- 미완료 취소/반품/교환/문의 유지
- 처리완료/철회/거부/답변완료 항목 제거
- 같은 주문번호 중복 집계 없음
- G마켓/옥션 제외
- 수집기 생존신호가 1분 간격으로 갱신
