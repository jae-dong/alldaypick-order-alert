FINAL v7.7.17 · 마켓 상품행·금액 직접복구

1. 쿠팡
- orderPrice에서 공식 할인액 discountPrice를 1회 차감해 실제 결제기준 상품금액으로 계산
- 부분취소/취소보류 수량 비율 반영 유지
- 할인 전 금액이 오늘 매출에 들어가던 과다 집계 수정

2. 스마트스토어
- 조건형 오늘 주문조회가 data 배열로 직접 응답하는 경우도 전부 인식
- contents/content/items/productOrders/productOrderList 응답 구조 지원
- 문자열 형태 상품주문번호도 상세조회에 포함
- 선물하기 미수락 주문 제외 유지

3. 11번가
- XML에서 상품행이 부모 노드에 있고 주소/상태가 자식 노드에 있을 때 부모 상품행이 버려지던 문제 수정
- 필드명 대소문자/구분자 차이를 보정해 주문번호·상품순번·금액 인식 확대
- 상태조회에서 기존 캐시에 없던 추가 상품행을 발견하면 주문문서로 복구
- 날짜 필드가 없을 때 주문번호의 YYYYMMDD를 상품주문 날짜로 보조 사용

4. 롯데온
- 주문/결제 금액이 중첩 객체에 들어오는 응답까지 재귀 탐색
- finalPayAmt, totOrdAmt, orderAmount, paymentAmt 등 금액 필드 확대
- 출고지시와 배송진행 병합 시 확인된 상품금액 유지

5. 공통
- 주문 건수는 송장수가 아닌 실제 상품주문 행 기준
- 동일 상품주문 행만 중복 제거
- G마켓·옥션은 API 승인 전까지 미연동·집계 제외
- 신규/발송대기/취소/반품/교환/문의 상태 로직과 텔레그램 썸네일 유지
