FINAL v7.7.14 · 마켓 공식 API 직접검증

핵심 원칙
- 샵모아 수치는 프로그램 집계나 상태판정에 사용하지 않습니다.
- 쿠팡·스마트스토어·11번가·롯데온의 공식 API 응답을 각각 직접 조회해 상태와 건수를 계산합니다.
- G마켓·옥션은 API 승인 전이므로 계속 미연동·집계 제외입니다.
- 스마트스토어 선물하기 미수락 주문은 신규·오늘 주문·매출·텔레그램에서 제외합니다.

쿠팡
- 주문 API의 상태별 조회 결과를 직접 기록합니다.
- 취소·반품·교환·문의 API를 서로 분리해 직접 조회합니다.
- 교환요청은 교환 API의 공식 상태 RECEIPT/PROGRESS를 최우선으로 판정합니다.
- 교환품 배송완료 표시만으로 진행 중 교환을 임의 종료하지 않습니다.
- 시작/정밀수집 때 진행 중 교환을 90일 범위에서 상태별로 조회합니다.

스마트스토어
- 상품주문 상세와 현재 클레임을 직접 대조합니다.
- 선물하기 WAIT_FOR_RECEIVING은 집계에서 제외합니다.
- 상품문의와 고객문의를 각각 공식 API로 직접 확인합니다.
- 429 제한 또는 조회 생략 때 오래된 문의 캐시를 현재 문의 건수로 계속 표시하지 않습니다.

11번가
- 주문 조회와 주문상태/클레임 조회 결과를 별도로 기록합니다.
- 주문번호와 상품행을 직접 대조해 취소·반품·교환 현재 상태를 반영합니다.

롯데온
- 출고지시 API와 배송진행 API를 모두 직접 조회합니다.
- 동일 상품행은 배송진행 상태를 출고지시 상태보다 우선합니다.
- 배송진행 응답의 상태문구가 비어 있어도 진행 API에서 발견된 행을 발송대기로 되돌리지 않습니다.

화면 집계
- 오늘/이번 달 주문 건수는 각 마켓의 실제 처리 단위로 계산합니다.
  쿠팡: 배송묶음, 스마트스토어: 상품주문번호, 11번가/롯데온: 상품행
- 매출은 같은 주문 총액이 여러 상품행에 중복 합산되지 않도록 주문 단위 금액을 유지합니다.
- G마켓·옥션은 연결 전까지 모든 합계에서 제외합니다.

직접검증 파일
- PC 수집기가 공식 API 조회 결과를 backend\market-direct-audit.json에 저장합니다.
- 프로젝트 폴더의 마켓직접검증_결과열기.cmd로 열 수 있습니다.
- API 키, 비밀키, 구매자 이름, 전화번호, 주소는 기록하지 않습니다.

기존 유지 기능
- 신규주문·취소·반품·교환·문의 텔레그램 알림과 상품 썸네일
- 오늘 판매 TOP 20 및 상품금액 보강
- 전체 글자 확대와 모바일/폴드폰 가독성
