올데이픽 주문알림 v50 대시보드 강제적용

v49가 안 보였던 원인
- 이전 UI 패치 구조에 의존했습니다.
- 휴대폰 캐시가 이전 화면을 계속 보여줄 수 있었습니다.

v50
- 이전 UI와 상관없이 새 대시보드를 직접 생성
- 오늘 주문/오늘 매출/이번달 총주문/이번달 총매출
- 처리 알림을 쇼핑몰별 현황 위에 표시
- 쇼핑몰별 수치는 오늘 기준
- 기존 중복 카드 강제로 숨김
- 캐시 버전 강제 변경

적용
1. 압축 해제
2. apply-v50.cmd와 apply-v50.ps1을 프로젝트 폴더에 복사
3. PowerShell 실행:
   powershell -NoProfile -ExecutionPolicy Bypass -File ".\apply-v50.ps1"
4. GitHub Desktop Summary:
   v50 대시보드 강제적용
5. Commit to main → Push origin

휴대폰에서 한 번만
- 사이트 열기
- 주소창 왼쪽 설정 아이콘
- 쿠키 및 사이트 데이터 삭제
- 사이트 다시 접속

화면만 수정하므로 npm run agent 재시작은 필요 없습니다.
