올데이픽 주문알림 FINAL v7.4.0 REVIEWED

[PC 수집기]
1. 프로젝트 최상위 폴더의 START_AGENT.cmd를 실행합니다.
2. 검은 창은 주문 수집 중 계속 켜 둡니다.
3. 이미 실행 중이라는 메시지가 나오면 기존 검은 창을 먼저 닫습니다.
4. 창을 강제로 닫아 잠금이 남은 경우 STOP_AGENT.cmd를 한 번 실행한 뒤 다시 시작합니다.

[수집 흐름]
- 신규주문이 주문확인/상품준비 상태가 되면 신규에서 빠지고 발송대기에 표시됩니다.
- 발송대기가 배송중/배송완료/구매확정으로 바뀌면 발송대기에서 빠집니다.
- 취소·반품·교환·문의는 주문 상태와 별도 사건으로 관리합니다.
- 미처리 사건만 남고 처리완료·철회·거부·답변완료 사건은 표시하지 않습니다.
- 동일 주문번호와 동일 상품 줄은 최신 상태 한 건만 사용합니다.
- G마켓과 옥션은 연결 상태가 true일 때만 집계합니다. 현재 미연결 상태에서는 제외됩니다.

[처음 적용한 날]
1. START_AGENT.cmd를 실행합니다.
2. 시작 정밀 동기화가 끝날 때까지 검은 창을 유지합니다. API 지연 시 수 분 걸릴 수 있습니다.
3. 웹앱의 지금 수집을 한 번 누릅니다.
4. 3~10분 뒤 웹앱을 새로고침합니다.
5. 신규/발송대기/취소/반품/교환/문의와 오늘 주문·매출을 마켓 관리자 화면과 비교합니다.

[주의]
- backend/.env.local에는 쇼핑몰 API와 Firebase 설정이 있으므로 외부에 공유하지 마세요.
- backend/node_modules는 GitHub에 올리지 않습니다.
- 검은 창의 npm warn은 실행 실패 문구가 아닙니다. ERROR 또는 실패 문구가 반복될 때 캡처합니다.
