올데이픽 주문알림 CLEAN v1.1.0

원본 전체 검사 결과
- 실제 루트 index.html이 1,288바이트로 축소되어 있었습니다.
- 정상 대시보드는 삭제되고 주문 메모창 내부 HTML 조각만 남아 있었습니다.
- 이전 버전, 백업, 패치, 설치 폴더가 프로젝트에 대량 누적되어 있었습니다.
- v1.0.7의 메모창 수정은 실제 detailDialog가 아닌 존재하지 않는 memoModal을 대상으로 했습니다.

이번 정리
- 마지막 정상 전체 대시보드 HTML 복원
- 깨진 루트 index.html 폐기
- 실제 주문 상세 dialog만 정상 모바일 팝업으로 정리
- 접속 시 모든 dialog 강제 닫힘
- 메모 저장 후 상세창 자동 닫힘
- 오늘 출고 필요 / 중요 주문 / 처리 완료 카드 제거
- 상세창의 중요 주문 / 수동 처리완료 버튼 제거
- 구버전·백업·패치 폴더를 새 패키지에서 모두 제외
- 기존 backend/.env.local은 설치할 때 자동 보존
- 서비스워커 새로 작성 및 이전 캐시 제거
- 화면 버전 CLEAN v1.1.0

적용
1. 압축을 풉니다.
2. ALLDAYPICK_CLEAN_1_1_0 폴더를 기존 프로젝트 폴더 안에 넣습니다.
3. INSTALL_CLEAN_1_1_0.cmd 실행
4. GitHub Desktop Summary:
   CLEAN v1.1.0 원본전체정리
5. Commit to main → Push origin
6. GitHub Pages 배포 완료 후 휴대폰 사이트 데이터 삭제
7. 사이트 재접속
8. PC 수집기는 검은창 Ctrl+C 후 backend에서 npm install, npm run agent

정상 확인
- 화면 상단에 CLEAN v1.1.0 표시
- 접속 시 대시보드가 바로 표시
- 주문 행을 눌렀을 때만 주문 상세창 표시
- 메모 저장 또는 닫기를 누르면 상세창 종료
