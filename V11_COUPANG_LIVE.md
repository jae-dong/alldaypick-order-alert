# v11 쿠팡 실제 신규주문 자동수집

## 동작 방식

- GitHub Actions가 10분마다 실행됩니다.
- 쿠팡 발주서 목록 조회 API에서 최근 30분의 `ACCEPT(결제완료)` 주문을 조회합니다.
- 조회 구간을 30분으로 겹치게 잡아 예약 실행이 조금 늦어져도 주문을 놓치지 않습니다.
- Firestore 문서 ID를 `coupang-주문번호-옵션ID`로 고정하여 중복 저장을 막습니다.
- 새 주문만 Firestore `orders` 컬렉션에 저장합니다.
- 휴대폰 두 대의 웹앱에서 실시간으로 같은 주문이 표시됩니다.

## 확인 방법

1. GitHub Desktop에서 v11 파일을 덮어쓰고 Commit → Push합니다.
2. GitHub Actions → Order Poller → Run workflow를 실행합니다.
3. 실행 결과가 Success인지 확인합니다.
4. 주문알림 사이트를 새로고침합니다.
5. 최근 30분 안에 쿠팡 결제완료 주문이 있다면 자동 표시됩니다.

## 오류 확인

Action이 실패하면 `poll-orders`를 눌러 마지막 빨간 단계의 메시지를 확인합니다.
API 키 권한이 신규 발급 상태라면 쿠팡 내부 승인에 시간이 걸릴 수 있습니다.
