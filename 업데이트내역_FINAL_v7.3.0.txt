올데이픽 주문알림 FINAL v7.3.0

핵심 수정
1. 신규주문/발송대기를 상품 줄 수가 아닌 주문번호 기준으로 1건씩 집계합니다.
2. 신규주문이 발송대기로 바뀌면 신규에서 빠지고 발송대기에만 남습니다.
3. 배송중/배송완료/구매확정으로 바뀌면 신규·발송대기에서 모두 빠집니다.
4. PC 수집기 시작 시 쿠팡의 전체 주문 상태를 정밀 재조회하여 과거 상태를 정리합니다.
5. 지금 수집 실행 시에도 전체 상태 정밀 동기화를 수행합니다.
6. 취소·반품·교환·문의는 주문 진행 상태와 별도로 관리합니다.
7. 미완료 클레임/문의만 남고 처리완료·철회·거부·답변완료 건은 표시하지 않습니다.
8. G마켓·옥션은 미연결 상태이면 모든 집계에서 제외합니다.

적용
- 기존 GitHub Pages 저장소 폴더에 이 파일들을 덮어씁니다.
- GitHub Desktop에서 Commit to main → Push origin을 진행합니다.
- 기존 수집기 창을 닫고 START_AGENT.cmd를 다시 실행합니다.
- 첫 시작 정밀 동기화는 평소보다 오래 걸릴 수 있습니다.
