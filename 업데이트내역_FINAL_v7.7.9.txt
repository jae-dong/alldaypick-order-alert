FINAL v7.7.9 · 전체 마켓 텔레그램 상품 썸네일 보강

1. 쿠팡
- 상품조회 API에서 파일명만 내려오는 vendorPath보다 실제 CDN 경로인 cdnPath를 우선 사용합니다.
- vendor_inventory 경로를 텔레그램 전송 가능한 쿠팡 원격 썸네일 주소로 변환합니다.
- REPRESENTATION/MAIN 대표 이미지만 선택하고 상세설명·배송안내·공지 배너는 제외합니다.

2. 스마트스토어
- 채널상품번호와 원상품번호를 모두 사용해 공식 상품 API 대표 이미지를 조회합니다.
- 그룹상품 전환·상품번호 변경 응답에 포함된 새 상품번호를 찾아 다시 조회합니다.
- API 이미지가 없을 때 공개 상품페이지의 OG/JSON-LD 대표 이미지를 한 번 더 확인합니다.

3. 롯데온
- 공식 상품 상세조회 API 응답의 대표이미지 필드명을 확대 적용했습니다.
- 단품/상품 이미지 목록의 대표·메인 이미지를 우선 선택합니다.
- API 이미지가 없을 때 롯데온 공개 상품페이지 대표 이미지를 확인합니다.

4. 11번가·G마켓·옥션
- 정상 작동 중인 11번가 로직은 유지하면서 OG/JSON-LD 추출을 보강했습니다.
- G마켓·옥션 주문에 상품번호 또는 이미지 주소가 들어오면 공개 상품페이지 대표 이미지까지 찾도록 추가했습니다.
- 현재 프로젝트에서 G마켓·옥션 주문 API 자체는 아직 연결 대기 상태이므로 실제 주문이 수집된 뒤 최종 확인이 필요합니다.

5. 텔레그램 이미지 전송
- 마켓별 Referer를 적용하고, 실패 시 원본 도메인/Referer 없음 방식으로 재시도합니다.
- Content-Type이 잘못 내려와도 JPG·PNG·GIF·WEBP 파일 헤더를 확인해 정상 이미지면 업로드합니다.
- 사진 전송이 완전히 실패하면 잘못 저장된 이미지 캐시를 삭제하여 다음 주문에서 다시 조회합니다.
- 이전 실패 캐시는 v9 캐시로 교체해 재사용하지 않습니다.
