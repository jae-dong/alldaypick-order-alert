FINAL v7.7.16 · 오늘 상품주문·금액 누락 복구

이번 버전의 목적
- 진행상태는 정상인 v7.7.15 기준을 유지합니다.
- 오늘 주문 40건처럼 일부 상품주문이 화면 집계에서 빠지던 문제를 복구합니다.
- 금액이 0원이거나 주문 총액이 상품행마다 반복되어 과다 집계되던 문제를 보강합니다.

주문 건수 고정 기준
- 송장번호 수, 배송묶음 수, 구매자 수가 아닙니다.
- 실제 상품주문 행 1개를 주문 1건으로 계산합니다.
- 한 구매자가 서로 다른 상품 2개를 합배송하고 송장 1개를 사용해도 2건입니다.
- 동일 상품주문 행이 반복 수집된 경우에만 중복 제거합니다.
- 상품 한 행의 수량이 2개여도 주문 건수는 1건입니다.

오늘·월 집계 누락 복구
- 기존 datetime 월 구독을 유지합니다.
- datetime이 비어도 이번 달 공식 API에서 다시 확인된 문서는 syncedAt 보조 구독으로 불러옵니다.
- 공식 API 직접검증 결과의 orderLines도 실제 주문문서와 ID 기준으로 병합하여 화면 집계 누락을 보완합니다.
- metricDate, orderDate, paymentDate 등 마켓별 날짜 필드를 모두 사용합니다.

쇼핑몰별 보강
- 쿠팡: 부분취소·취소보류 수량이 있는 상품은 원 주문수량 대비 실제 유효수량 비율로 상품금액을 계산합니다.
- 스마트스토어: 오늘 00시부터 PAYED_DATETIME/ORDER_DATETIME 조건을 여러 방식으로 조회하여 상품주문번호를 재발견합니다.
- 11번가: 주문상태 상세 응답에서 상품명·옵션·수량·상품행 금액·주문일자를 다시 보강하고, 옵션이 다른 상품행이 합쳐지지 않게 식별키를 확대합니다.
- 롯데온: 출고지시·배송진행 응답의 상품행 날짜와 상품행 금액 필드를 우선 보존합니다.

매출 계산
- 상품행 실제 금액이 있으면 상품행 금액을 합산합니다.
- 같은 주문 전체금액이 여러 상품행에 반복돼도 중복 합산하지 않습니다.
- 상품행 금액이 모두 없을 때만 주문 전체금액을 한 번 사용합니다.

고정 제외 기준
- 스마트스토어 선물하기 WAIT_FOR_RECEIVING 미수락 주문은 신규·오늘 주문·매출·텔레그램에서 제외합니다.
- G마켓·옥션은 API 승인 전까지 미연동·집계 제외입니다.

유지 기능
- 쿠팡·스마트스토어·11번가·롯데온 공식 API 직접검증
- 신규주문·취소·반품·교환·문의 텔레그램 상품 썸네일
- 오늘 판매 TOP 20 및 상품금액 보강
- 전체 글자 확대
- 쿠팡 교환 RECEIPT/PROGRESS 진행 건 표시

기타 수정
- 마켓직접검증_결과열기.cmd와 OPEN_MARKET_AUDIT.cmd를 Windows CMD 인코딩 오류가 나지 않는 ASCII/CRLF 방식으로 수정했습니다.
