FINAL v7.7.11 · 주문·CS 상태정합성 보강

1. 쿠팡
- 시작 시 신규·발송대기만 확인하던 빠른 수집은 그대로 유지했습니다.
- 빠른 시작 이후 백그라운드에서 신규, 발송대기, 배송지시, 배송중, 배송완료, 직접배송 상태를 모두 다시 조회합니다.
- 14일 범위의 공식 상태를 대조해 이전 상태에 남은 주문을 정리합니다.
- 오래된 RECEIPT 교환요청은 기록을 삭제하지 않고 현재 진행 숫자에서만 제외합니다.

2. 스마트스토어
- 네이버 공식 주문 상태를 그대로 사용하며 샵모아 표시를 맞추기 위해 임의로 신규 상태를 만들지 않습니다.
- 기존 취소·반품·교환·문의 텔레그램 연결과 썸네일 조회 로직을 유지합니다.

3. 11번가
- ordCnDts, 취소요청 여부, 반품요청 여부, 교환요청 여부 등 추가 필드를 인식합니다.
- claimType이 비어 있어도 주문 상태가 취소요청/취소완료이면 취소 이벤트를 생성합니다.
- 최초 접수된 취소·반품·교환도 텔레그램 대상으로 유지합니다.

4. 롯데온
- 매 수집마다 주문지시 API와 배송진행 API를 모두 확인합니다.
- 배송진행 목록에서 빠진 완료 주문은 현재 발송대기에서 자동 종료합니다.
- 최초 접수된 취소·반품·교환 텔레그램 연결을 유지합니다.

5. 화면 표시
- G마켓·옥션처럼 주문 API가 연결되지 않은 마켓은 0건 대신 미연동으로 표시합니다.
- 미연동 마켓은 오늘 주문·매출과 진행 주문 합계에서 제외되었음을 화면에 표시합니다.

6. 그대로 유지한 기능
- 쿠팡·11번가 상품 썸네일 전송
- 신규주문·취소·반품·교환·문의 텔레그램 중복방지
- 무료한도 보호와 로컬 캐시
- 모바일·폴드폰용 주문현황 표

7. 아직 필요한 외부 설정
- G마켓·옥션의 실제 주문·취소·반품·교환·문의 수집은 ESM Trading API 사용 승인, 인증키, 판매자 ID, 허용 IP 설정이 필요합니다.
- 해당 정보가 없는 상태에서는 프로그램이 데이터를 임의로 가져올 수 없으므로 미연동·집계 제외로 정확히 표시합니다.
