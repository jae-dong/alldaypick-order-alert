올데이픽 주문알림 CLEAN v58

수정 내용
- 접속 즉시 Firestore 실시간 리스너 시작
- 서버 확인을 기다리지 않고 저장된 주문 캐시 먼저 표시
- 새로고침, 재접속, 앱 복귀, 네트워크 복구 시 자동 재연결
- 지금 수집 버튼은 action=reconcile로 이번 달 정밀 동기화
- PC 수집기 상태 문서 읽기 실패 시 쇼핑몰 마지막 수집 시간으로 보조 판정
- 주문현황과 오늘분석은 캐시된 주문을 즉시 표시한 뒤 최신 데이터로 갱신
- 텔레그램은 신규주문, 주문취소, 반품요청, 문의사항만 전송
- 발송대기, 배송중, 배송완료, 교환요청 알림 차단
- 텔레그램 테스트 문구도 네 가지 알림으로 변경

문의사항 주의
현재 백엔드에 문의 API 수집기가 연결된 마켓만 문의 알림이 옵니다.
문의 데이터가 Firestore에 eventType=inquiry로 저장되면 텔레그램으로 전송됩니다.

적용
1. 압축 해제
2. V58_FINAL 폴더를 기존 프로젝트 폴더 바로 아래에 둡니다.
3. install-v58.cmd 실행
4. GitHub Desktop Summary: v58 자동연결 텔레그램필터
5. Commit → Push
6. 검은 창 Ctrl+C
7. 원래 프로젝트 backend에서 npm install
8. npm run agent
9. 휴대폰 사이트 데이터 한 번 삭제 후 재접속
10. 지금 수집을 눌러 이번달 정밀 동기화

Firestore 규칙은 v57 규칙을 이미 게시했다면 다시 게시하지 않아도 됩니다.
