올데이픽 주문알림 CLEAN v57 전체 데이터 정합성 수정

핵심 수정
- PC 수집기 상태 실패 원인이었던 Firestore 규칙 수정
- 텔레그램 테스트 명령 허용
- 임시 고정 주문·매출·처리 건수 완전 제거
- 실제 Firestore 주문 데이터만 계산
- G마켓·옥션은 연결 전 계산에서 제외
- 향후 연결되면 연결일 이후 주문부터 합산 가능
- 취소·반품·교환·문의는 실제 미처리 건만 표시
- 완료·철회·종결 또는 처리완료 버튼 적용 시 자동 제외
- 주문 목록은 최근 31일 주문을 기본 표시
- 오늘 분석 항상 표시
- 지금 수집은 이번달 1일부터 쿠팡·스마트스토어·11번가·롯데온 정밀 동기화
- 발송대기로 처음 발견된 주문도 텔레그램 전송
- 텔레그램 15초 제한, 최대 3회 재시도
- 한 쇼핑몰 오류가 다른 쇼핑몰 수집을 막지 않음

적용
1. 압축 해제
2. V57_FINAL 폴더를 프로젝트 폴더 바로 아래에 둡니다.
3. install-v57.cmd 실행
4. GitHub Desktop: v57 전체데이터 정합성 수정
5. Commit → Push
6. Firebase 콘솔 Firestore Database → 규칙에서 firestore.rules.txt 내용 붙여넣기 → 게시
7. 검은 창 Ctrl+C
8. 기존 프로젝트 backend에서 npm install
9. npm run agent
10. 텔레그램 테스트
11. 지금 수집을 눌러 이번달 정밀 동기화

중요
Firebase 규칙 게시가 빠지면 PC 수집기 상태와 텔레그램 테스트 결과 확인이 계속 실패합니다.
각 판매자 API는 사용자 PC 허용 IP가 필요하므로 실제 정밀 조회는 사용자 PC에서 실행됩니다.
